Essas palavras-chave também são consideradas reservadas em atributos (excluindo listas de argumentos de atributos), mas algumas implementações as tratam da mesma forma que as outras.(desde C++11)

Além das palavras-chave, existem _identificadores com significado especial_, que podem ser usados como nomes de objetos ou funções, mas têm significado especial em certos contextos.

Além disso, todos os [identificadores](<#/doc/language/name>) que contêm um sublinhado duplo `__` em qualquer posição e cada identificador que começa com um sublinhado seguido por uma letra maiúscula são sempre reservados, e todos os identificadores que começam com um sublinhado são reservados para uso como nomes no namespace global. Veja [identificadores](<#/doc/language/name>) para mais detalhes.

O namespace `std` é usado para colocar nomes da biblioteca padrão C++. Veja [Estendendo o namespace std](<#/doc/language/extending_std>) para as regras sobre como adicionar nomes a ele.

O nome `posix` é reservado para um futuro namespace de nível superior. O comportamento é indefinido se um programa declarar ou definir algo nesse namespace. | (desde C++11)
